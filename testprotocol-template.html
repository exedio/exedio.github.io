exedio persistence

Persistence Framework for Java.

Test Environments

Each build of exedio persistence is subject to extensive automated unit testing on various environments. An environment is the combination of the Database Management System (DBMS), the JDBC driver and the Connect Properties. The following table lists all environments, the framework is tested on.

@TABLE@
Environment DBMS JDBC driver Connect Properties
Name Version (i.j) Name Version (i.j)

The exedio console shows your environment along with the list of enviroments, your version of exedio persistence has been tested on.

You may want to try the tests on your own environment, to make sure, that your project's combination of DBMS, JDBC driver and Connect Properties works correctly. For your convenience, the junit tests of the framework are distributed in an extra jar: exedio-cope-test.jar. This jar contains a test suite com.exedio.cope.CopeTest, that you may execute to verify your environment. Please note, that the demoshop already has an ant target copetest for this purpose.
